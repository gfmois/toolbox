Toolbox CLI

Ship and run internal CLI products as plugins — from a single binary

Toolbox es un repositorio para empaquetar y distribuir productos internos de línea de comandos como plugins dentro de un único binario. Está pensado para simplificar el despliegue, centralizar actualizaciones y permitir que nuevos comandos se agreguen sin fricción.

¿Qué es?

Una plataforma CLI modular que carga funcionalidades como plugins. Cada plugin expone comandos específicos, y el binario principal actúa como el punto de entrada unificado.

¿Cómo funciona?

El binario central descubre plugins instalados, registra sus comandos y ejecuta la lógica correspondiente. Esto permite versionar y publicar herramientas internas de manera consistente.

¿Para quién es?

Equipos que necesitan distribuir múltiples utilidades internas sin mantener múltiples instaladores o rutas de actualización independientes.

Instalación

Asegúrate de tener permisos de acceso al repositorio y a los artefactos internos. Luego ejecuta la instalación del binario principal y configura los plugins requeridos. 

# Ejemplo conceptual
# 1) Instalar el binario principal
# 2) Instalar plugins autorizados
# 3) Verificar comandos disponibles

toolbox --version
toolbox plugins list

Actualizar

Las actualizaciones se realizan desde un único punto. Esto reduce la complejidad de mantener varias herramientas en paralelo. 

# Actualiza el binario principal y plugins asociados

toolbox self update
toolbox plugins update --all

Uso básico

Ejemplos típicos de interacción con la CLI:

# Ver ayuda general
toolbox help

# Ejecutar un comando de un plugin
# (el nombre del plugin y comando pueden variar)
toolbox   [args]

# Ver comandos disponibles
toolbox list

Arquitectura (alto nivel)

Core

Orquesta la carga de plugins, el registro de comandos y la ejecución.

Plugins

Paquetes independientes que aportan comandos y funcionalidades específicas.

Distribución

Un único binario simplifica el despliegue y el control de versiones.

Buenas prácticas

  • Versiona plugins de forma semántica para facilitar compatibilidad.
  • Documenta cada comando con ejemplos claros y resultados esperados.
  • Mantén el core estable y enfoca cambios frecuentes en los plugins.
